Architecture

Snapline is an npm monorepo. Consumers install one package; maintainers split concerns across five published libraries plus a private demo workspace.

Package layers

Snapline Hub (optional)

Snapline Hub is a separate, optional reporting UI that stores and visualizes TestRunReport data. It is not part of the npm packages — use it when you want a centralized dashboard for test run history. Push reports via pushTestReportToHub() or SNAPLINE_HUB_URL. The Python edition has equivalent push_test_report_to_hub().

Runtime flow — testSuite

testSuite(name, config) │ ├─ auth.initialize() optional OAuth2 / OpenID / Basic │ ├─ api executeApi → snapline vs expectedFile ├─ dbComparison fetch source + target rows → snapline ├─ apiToDb executeApi → fetch DB row → snapline ├─ dbToApi fetch DB row → executeApi → snapline └─ publishAndPoll publish queue → poll DB / file / topic → snapline │ ▼ snapline(live, expected, { ignoreFields, transformations, dataMapping }) │ ▼ { match, diff, processed, expected } → TestSuiteResult

Fixture case runners

Two runners live in core for data-driven scenarios:

  • runApiFixtureCases — HTTP per case, compare to expected.json
  • runSnaplineFixtureCases — offline: compare live.json vs expected.json

Each case directory contains case.json (metadata) plus protocol-specific files. Path sandboxing ensures fixture paths cannot escape the fixtures root.

Published package dependencies

@vaagatech/snapline-core has zero external runtime dependencies. It exports the DbConnectionLike interface for SQL — you bring your own Postgres/MySQL driver. SQLite and stub DB helpers live in @vaagatech/snapline-demo-shared (demo only, not published to npm consumers as a required dependency).

Security & reliability

  • SSRF protection — assertSafeUrl blocks metadata hosts and optional private networks
  • HTTP timeouts — 30s default on all protocol executors
  • Path sandboxing — fixture and JSON file loads constrained to allowed roots
  • Report redaction — redactFields masks sensitive step data in HTML/JSON reports

Demo Scenarios

The repo ships 19 integration scenarios with a local mock REST/GraphQL/SOAP server and SQLite seeds. Use them as runnable reference implementations.

Run commands

npm install
npm run demo                    # build + run all 19
npm run demo:list               # list ids and infra needs
npm run demo:run -- api-vs-file-graphql
npm run demo:run -- --build snapline-transformations

Infra is auto-wired by demo/scripts/run-scenario.mjs:

  • needsServer → starts mock API, sets API_BASE_URL, OAuth credentials
  • needsDatabase → seeds temp SQLite, sets APP_DATABASE_URL, etc.

Scenario folder structure

Each scenario is a standalone package you can copy into your project:

demo/scenarios/api-vs-file-graphql/
├── fixtures/cases/...
├── src/
│   ├── run.ts        # entry point
│   ├── env.ts        # env vars + report finalization
│   ├── auth.ts       # OAuth from env
│   └── demo-data.ts  # transforms / mappings
└── package.json      # only @vaagatech/snapline-core

Run against your own API

cp demo/scenarios/api-vs-file-graphql/.env.example demo/scenarios/api-vs-file-graphql/.env
# edit API_BASE_URL, CLIENT_ID, CLIENT_SECRET
npm run start --workspace=@vaagatech/snapline-demo-scenario-api-vs-file-graphql

HTML reports from demo

REPORT_FORMAT=html REPORT_OUTPUT=./reports/demo.html npm run demo
npm run start:report:html --workspace=@vaagatech/snapline-demo-run-all
